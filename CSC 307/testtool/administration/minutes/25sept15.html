  2. Talk to the customer

    • Test tool generate class results and be downloaded by teacher?
           Depends on type of test - some would require grading. If all multiple choice, it would be an option, like send it in an email. At least review from within the application.

    • Subject field?
           Professor defines subjects and assigns them to questions.

    • Support having multiple users answering the same test?
           Everyone looks at one computer, put in their usernames and it gets graded all together.

    • Save the tests / how long make them available?
           Don't worry about it. Professor will manage question bank.

    • Workflow for creating test?
           Having different types of generation (basic, advanced, etc). Make a question bank. Professor will go into the question bank and choose options (how long, which class, level of difficulty, time for whole test, keywords). You can also have it say 40% level 5, 20% long answer, 10% short answer, etc. How good the test generator is (correct average rating, time, etc). Smart about the ordering of a test (start with MC, short answer, long, etc in order). Options for how the test is going to be laid out.

    • Time limits for specific questions?
           Implementation could be beneficial, but not essential.

    • Immediate feedback when completing practice tests?
           Yep. As long as none of the questions require grading. If it does, we can use something like: "You got 10/12 on MC, rest of grade is pending instructor review."

    • Test display multiple questions at a time vs single questions each page?
           Layout is up to you. We can do single questions with pagination support, progress bar at the end.

    • TIP: UI design layout.
           Get going on design and show to customer (ask if they like or not)?

    • Results page for student.
           Instructor sees open tests and progress that students have with it. Results page for what the student has taken.

    • Reports for tests.
           Click on name of the test, have windows open that show strengths, weaknesses.
