1.3. Operational Setting

The operational setting of the Test tool is to help teachers design tests and make it easier for them to keep track of different questions that were on past tests.

The Test tool is intended to be general public domain software. But is intended for use of the Computer Science Department at Cal Poly. The setting is suited for a school or any person that wants to create a test. Teachers should collaborate with each other to create questions.

As noted in the preceding section on system personnel, the primary customer for the Test Tool is the Computer Science Deparment at Cal poly. As such, the tool's requirements will reflect the point of view of the customer base, including their work setting. Given the intent to develop a general purpose product, the customer has made a conscience effort to exclue setting-specific features from the Test Tool requirements.

The Test Tool has been designed to fit the curriculum used by Gene Fisher in undergraduate software engineering courses at Cal Poly University. The course is one quarter long that cover standard aspects of software engineering, with enphasis on the practical application of formal methods. Other instructors may find the Test Tool and its development useful as course examples.
